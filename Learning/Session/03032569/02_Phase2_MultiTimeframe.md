# Phase 2 – Multi-Timeframe Execution Model

## Learning Objectives
- อธิบายหลักการ Top-Down Analysis และเหตุผลที่ต้องเริ่มจาก HTF
- สร้างกฎการเข้าออกที่สอดคล้องระหว่าง HTF Bias กับ LTF Trigger
- ใช้กฎพฤติกรรม (เช่น ห้าม FOMO/แก้มือ) เพื่อรักษาความมีวินัยระหว่างเทรด

## Introduction
เมื่อเข้าใจโครงสร้างตลาดแล้ว ขั้นต่อมาคือการประสานหลายกรอบเวลาให้สื่อสารภาษาเดียวกัน Top-Down Analysis ลดความเสี่ยงจากการเห็นสัญญาณ LTF ที่สวนกับภาพใหญ่ ช่วยให้ทุกคำสั่งยึดหลักฐานเดียวกัน

## Core Concepts
1. **Top-down vs Bottom-up** – การเริ่มวิเคราะห์จาก HTF ทำให้มองเห็น sentiment และระดับสำคัญก่อน แล้วจึงเลื่อนลง LTF หาจุดเข้า การเริ่มจาก LTF มีความเสี่ยงสูงเพราะมักฝืนโครงสร้างใหญ่และปรับมุมมองให้เข้ากับสัญญาณที่เห็น (confirmation bias)[3]
2. **การเลือก Timeframe Combo** – เลือกเฟรมให้ตอบโจทย์สไตล์ เช่น Bias (H4), Structure (H1), Setup Zone (M30), Trigger (M15) และควรยึดชุดเดิมอย่างน้อย 30–50 เทรดเพื่อสร้างสถิติ[3]
3. **เงื่อนไขการเข้า** – ราคาแตะโซนที่เตรียมไว้, โครงสร้างย่อยสอดคล้อง Bias, มีสัญญาณยืนยัน (CHOCH/BOS) เพื่อหลีกเลี่ยงการเข้าแบบ random
4. **กฎพฤติกรรม** – กำหนดไว้ล่วงหน้าว่าห้ามเข้าเพราะกลัวพลาด ห้ามแก้มือหลังขาดทุน และต้องเช็ก checklist ให้ครบก่อนกดคำสั่ง

## Practical Example
1. วิเคราะห์ H4 เพื่อหาทิศทางหลัก สมมติ bearish
2. ไป H1 เพื่อดูว่าราคา retrace ขึ้นไหนคือ LH ล่าสุด
3. บน M30 มาร์ก Supply Zone ที่ก่อให้เกิด impulse ลง
4. เมื่อราคากลับมาที่โซน ให้รอ M15 สร้าง CHOCH ลงก่อน entry
5. วาง SL เหนือ wick sweep ล่าสุดและตั้ง TP ที่โซน liquidity ฝั่งตรงข้าม

## Common Mistakes
- กระโดดระหว่างชุด timeframe ทำให้แผน execution ขาดความต่อเนื่อง
- ลืมตรวจ HTF ก่อนเห็นสัญญาณ LTF สวย ๆ แล้วเข้าแบบสวนเทรนด์
- ไม่มี check-list ทำให้ตัดสินใจตามอารมณ์เมื่อราคาใกล้โซน

## Advanced Insight (Optional)
สามารถเสริม confluence ด้วย Fair Value Gap, Order Block หรือ Volume Profile บน HTF เพื่อหาโซนคุณภาพสูงกว่าเดิม

## Summary
Phase 2 ทำให้ทุกการตัดสินใจไหลตามกระบวนการเดียวกัน เริ่มจาก HTF หา Bias แล้วค่อยลดเฟรมเพื่อหา execution ที่แม่นขึ้น เมื่อกำหนดกฎและพฤติกรรมชัดเจน จะลดการเทรดด้วยอารมณ์ได้มาก
